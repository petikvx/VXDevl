Inputs for Antivirus Neural Networks

1        File operations:

1.1    Name: Replication (write/append) pattern

Assumption:

A virus modifies other files during propagation. The child files will be executed later and infect other files.

 

Description:

A virus usually propagates itself by infecting other files. Recording the genealogy of file writing, we may identify a virus by a file with a lot of children or grandchildren.

 

Overhead:

Hooks on file APIs, log files stored with security guarantee, analysis

 

Complexity:

Moderate

1.2    Name: Deletion pattern

Assumption:

A virus deletes some useful files or some temporary files generated by the virus.

 

Description:

Some viruses remove a lot of files automatically. If such removal is not prompted by a user, it may be suspicious. Or it will remove some intermediate files generated by it.

 

Overhead:

Hooks on file APIs, log files stored with security guarantee, analysis

 

Complexity:

Moderate

 

2        Network operations:

2.1    Name: IP address pattern

Assumption:

Some worms propagate over network. They will access IP addresses in some specific pattern.

 

Description:

Some worms attack other computer by randomly chosen destination IP addresses, sequentially increasing addresses, or pre-chosen IP list. Also it may cause a lot of pending connections.

 

Overhead:

Hooks on network APIs or network device monitoring, status checking on connections, log files stored with security guarantee, analysis

 

Complexity:

Difficult

2.2    Name: Port scan

Assumption:

A worm tries to find victims by scanning ports on remote hosts.

 

Description:

Some worms attack a specific computer by scanning a lot of ports on that computer. They send probing messages to a lot of ports on the victim host.

 

Overhead:

Hooks on network APIs or network device monitoring, log files stored with security guarantee, analysis

 

Complexity:

Moderate

 

2.3    Name: Repetitive payload

Assumption:

A malicious program will send the same content over the network repetitively.

 

Description:

Some viruses/worms attack other computers by sending a lot of common data.

 

Overhead:

Hooks on network APIs or network device monitoring, logging signatures, analysis

 

Complexity:

Difficult

 

2.4    Name: Packet interception

Assumption:

A Trojan horse will try to intercept secret information in packets.

 

Description:

Some Trojan horse snoops personal information by monitoring secret information like passwords sent over network.

 

Overhead:

Hooks on memory buffer read, distinguishing a legal monitor from a snooper

 

Complexity:

Unknown

 

2.5    Name: Automatic Replicated Emails

Assumption:

Many malicious programs will propagate by emails.

 

Description:

Some viruses will automate email sending procedure and put some common attachments inside. And contagious attachment contains code to send emails.

 

Overhead:

Monitor emails with same content and for different recipients, hooks on email sending APIs

 

Complexity:

Difficult

 

3        Kernel modifications:

3.1    Name: Driver registration

Assumption:

Some virus will behave like a driver.

                                                                                                                            

Description:

Some viruses register them as a driver, keep resident in memory, and execute kernel instructions.

 

Overhead:

Hooks on driver registration APIs

 

Complexity:

Easy

 

 

3.2    Name: Interrupt interception

Assumption:

Some viruses will add a hook to system interrupts.

 

Description:

Some viruses monitor keyboard/display events and snoop for special information.

 

Overhead:

Checking if there are hooks on interrupts

 

Complexity:

Unknown

3.3    Name: System configurations

Assumption:

A virus may change the system configurations. Here we just look at registry.

 

Description:

Some viruses modifies system registry.

 

Overhead:

Monitoring writes to system registry.

 

Complexity:

Easy

 

4        Boot modifications:

4.1    Name: Bios

Assumption:

A virus changes Bios data .

 

Description:

Some viruses modifies Bios .

 

Overhead:

Monitoring Bios operations

 

Complexity:

Easy

 

4.2    Name: Boot Sectors

Assumption:

A virus changes boot sectors, such as Master Boot Record

 

Description:

Some viruses modify MBR.

 

Overhead:

Monitoring writes to Master Boot Record

 

Complexity:

Easy

 

5        Resource usage:

5.1    Name:  CPU

Assumption:

A virus may have some pattern of CPU usage

 

Description:

Some viruses may have a special pattern using CPU, such as intermittent usage, or just making the CPU busy

 

Overhead:

Monitoring CPU usage, anomaly detection

 

Complexity:

Moderate

 

5.2    Name: Memory

Assumption:

A virus may use memory in some special pattern

 

Description:

Some viruses may have a special pattern using memory, such as replicated memory copying

 

Overhead:

Monitoring memory usage, anomaly detection

 

Complexity:

Moderate

5.3    Name: Disk

Assumption:

A virus may access disk in some special pattern

 

Description:

Some viruses may have a special pattern in accessing disk address, such as (de)fragmentation?

 

 

Overhead:

Monitoring disk usage, anomaly detection

 

Complexity:

Moderate

 

5.4    Name: Bandwidth

Assumption:

A virus may have some bandwidth usage pattern

 

Description:

Some viruses may have a special pattern using bandwidth, such as burst in bandwidth, intermittent usage of bandwidth

 

 

Overhead:

Monitoring bandwidth usage, anomaly detection

 

Complexity:

Easy

 

 

6        Special behaviors:

6.1    Name: Self-modification

Assumption:

Polymorphic virus may change its own code to avoid detection.

 

Description:

Some viruses may change their own code. So the code to be executed will be different than the code stored in the disk.

 

Overhead:

Modal checking: Address analysis

 

Complexity:

Moderate, and difficult to do it for dynamic contents.

 

6.2    Name: Self Encryption/Decryption

Assumption:

Encryption/decryption is done by some known procedure.

 

Description:

Viruses may encrypt and decrypt themselves to avoid detection. They decrypt their own code before execution and encrypt it before infection

 

Overhead:

Modal checking: encryption/decryption instructions

 

Complexity:

Moderate

6.3    Name: Code-like binary array

Assumption:

Program code looks different than usual data.

 

Description:

Malicious code may store some binary data which looks like normal information. Then, it will copy these data to code portion and jump into these code for further execution.

 

Overhead:

Check static contents of data arrays.

 

Complexity:

Moderate, and difficult to do it for dynamic contents.

 

6.4    Name: Time-dependent branch

Assumption:

Many viruses will keep dormant until a specific time comes.

 

Description:

Some viruses put a trigger (like a due day) for executing the malicious part of the code

 

Overhead:

Branch analysis

 

Complexity:

Easy, but it may cause false positives

 

6.5    Name: General suspicious strings

Assumption:

Virus writers may get satisfaction by playing jokes or being sarcastic against other people.

 

Description:

Some viruses contain suspicious strings like “virus” or slang. We need a virus slang library for matching.

 

Overhead:

Static string matching, virus slang library

 

Complexity:

Easy

6.6    Name: Suspicious programming style

Assumption:

Virus writers may be not serious programs and don’t care the style is good or not. Also for them, it doesn’t matter if there are any bugs in the code.

 

Description:

Check the format of the code, comments, and other static features. Also check if there are a lot of run-time violations, such as segmentation faults.

 

Overhead:

Static parsing, run-time error monitoring.

 

Complexity:

Difficult
